### 2026-02-06 · 回溯组合问题（LeetCode 77）

**问题背景**
- 使用回溯算法生成组合
- 代码能跑，但对边界条件与剪枝逻辑理解不够确信

**输入代码特点**
- 回溯 + startIndex
- 使用剪枝表达式：`n - (k - len(path)) + 2`
- 缺少类型提示与边界防御

**Codex 关键洞察**
- 指出需要显式处理 `k < 0` 或 `k > n` 的非法输入
- 补充 `typing.List`，让函数签名更清晰
- 确认剪枝上界的数学含义，避免“看不懂但照抄”

**我的收获**
- 剪枝不是技巧，而是“剩余可选元素是否足够”的判断
- 好的 debug Prompt 能逼我把「为什么这么写」说清楚
